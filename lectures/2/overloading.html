Перегрузка операторов в C++#

Перегрузка операторов — это механизм, позволяющий использовать привычные математические и логические операторы для работы с пользовательскими типами данных. Это делает код более читаемым и понятным, а также позволяет использовать уже знакомые операции для работы с новыми типами данных.

Одним из примеров перегрузки операторов является оператор +=, который используется для сложения двух переменных. В контексте перегрузки операторов, оператор += может быть переопределён для работы с пользовательскими типами данных.

В данной лекции разберем механизм перегрузки оператора на пример класса рациональных числе(дробных). Пример взят с сайта нашей кафедры.

Класс рациональных чисел#

Добавим библиотеки:

#include <iostream>
using namespace std;

Начнем писать наше класс дробных чисел:

class Ratio {
    private:
        int a,b;
    public:
        explicit Ratio(int na=0, int nb=0) : a(na), b(nb) //Конструктор
        { 
            cout << "Hi: (" << a << "/" << b << ")" << endl; 
        }
        Ratio(const Ratio& src) : a(src.a), b(src.b) // Конструктор копирования
        { 
            cout << "Hi: (" << a << "/" << b << ")" << endl << flush; 
        }
        ~Ratio() // Деструктор
        { 
            cout << "RIP: (" << a << "/" << b << ")" << endl << flush; 
        }

Перегрузка оператора присваивания = :#

Ratio& operator=( const Ratio& src ) 
{ 
    a=src.a; 
    b=src.b; 
    return *this; 
}

В методах класса мы можем использовать ключевое слово this, который является указателем на объект класса, от которого он был вызван.

*this — разыменованный указатель, т.е. мы возвращаем объект класса.

В данном примере, мы делаем возврат класса, для того, чтобы у нас работало множественное присваивание:

Ratio r1(1,2), r2(3,7), r3(4, 9);

r3 = r1 = r2;

Note

Чему будут равны r1, r2, r3 в примере выше?

Приведение типов и ключевое слово explicit#

Еще в языке Си появилось так называемое приведение типов. Чаще всего оно встречается неявно и вы неоднократно сталкивались с этим:

double d = 5;

В данном примере целое число 5 неявно преобразуется в вещественный тип. Для явного приведения типов необходимо перед числом 5 в скобках указать необходимый тип данных:

double d = (double)5;

Но не всегда неявное приведение желательно, особенно когда вы только начали писать класс неявное приведение может вызвать большое количество ошибок. Для того чтобы запретить неявное приведение перед конструктором пишется ключевое слово explicit. Т.е. в нашем случае

Ratio new_R = 5; // Будет вызвано с ошибкой
Ratio new_R = (Ratio) 5; //Создастся объект new_R = 5/1

а если убрать ключевое слово explicit, то и первая строчка создаст объект new_R = 5/1. Почему 5/1 ? Т.к. в нашем конструкторе используются параметры по умолчанию и знаменатель по умолчанию равен 1.

Еще пример неявного приведение типа:

Ratio r(1, 3), r1;
r += 3; // цифра 3 неявно приводится к тип Ratio
        // r += (Ratio)3;
r = r1 + 8; // цифра 3 неявно приводится к тип Ratio
            // r = r1 + (Ratio)8;

Перегрузка оператора присваивания += :#

 Ratio& operator+=( const Ratio& src )
    { 
        a = a * src.b + b * src.a;
        b *= src.b;
        return *this;
    }

Вспомогательные методы и перегрузка оператора + :#

Напишем константные методы доступа:

int chislitel() const { return a; }
int znamenatel() const { return b; }

Почему методы константные смотрите ниже.

Операция переворота дроби:

Ratio& reverse() { int tmp=a; a=b; b=tmp; return *this; }

Явное поведение класса при преобразовании int:

operator int() const { return a/b; }

 Ratio operator+( const Ratio& src2 ) const
 { 
    Ratio res(*this); // создаем объект класса с помощью конструктора копирования
    res += src2;
    return res;
 }

Warning

Все методы выше описывают класс рациональных чисел. Дальнейшие Перегрузки операторов производятся вне тела класса.

Перегрузка оператора << :#

Рассмотрим как объявить программе вывод объекта класса с помощью cout

ostream& operator<<(ostream& os, const Ratio& src)
{ 
    os << "(" << src.chislitel() << "/";
    os << src.znamenatel() << ")" << flush;
    return os;
}

ostream — это класс в языке C++, который представляет потоки вывода. Он позволяет отправлять данные на устройства вывода, такие как консоль или файлы.

Объект типа ostream получает значения различных типов, преобразует их в последовательность символов и передаёт через буфер в определённое место для вывода. Теперь мы легко можем использовать обычный нам способ вывода в консоль:

Ratio r1(2,3);
cout << r1;

Note

flush в C++ — это функция для принудительного сброса буфера вывода. Она сигнализирует системе немедленно записать содержимое буфера в указанное место.

Вместо flush можно использовать endl, т.к. он уже содержит в себе flush

Note

Методы где после прототипа функции, но до ее тела написано ключевое слово const — называются константными. Их использование необходимо при вызове через константные ссылки объекта класса. Звучит и правда сложно, но более подробно зачем и почему будет описано ниже при описании перегрузки оператора <<.

Константные ссылки позволяют нам убедиться, что передаваемый по ссылке объект не будет изменен. Но за это мы платим тем, что не можем вызывать никакие методы этих объектов. Т.е. в нашем случае при перегрузке оператора мы вызываем методы для доступа к числителю и знаменателю. Эти методы вызываются к объекту переданный по константной ссылке. И дабы вызвать этим методы нам приходится отдельно указать, что эти методы являются константными, т.е.:

int chislitel() const { return a; }
int znamenatel() const { return b; }

Перегрузка оператора >> :#

istream& operator>>(istream& is, Ratio& r)
{
    int na, nb=1; 
    is >> na;
    char c; 
    is >> c;
    if (c=='/') is >> nb;
    r = Ratio(na,nb);
    return is;
}

istream - это базовый класс в C++ для работы с потоками ввода данных. Он определен в стандартной библиотеке в заголовочном файле iostream и предназначен для чтения данных из различных источников (консоль, файлы, сетевые интерфейсы и т.д.).
